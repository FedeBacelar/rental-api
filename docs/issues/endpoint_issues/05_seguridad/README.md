# 5. Seguridad

Por ahora no vamos a trabajar endpoints de seguridad.

El proyecto ya tiene una base de models relacionados con usuarios, roles y permisos, pero no vamos a desarrollar esa parte en esta etapa.

## Motivo

La seguridad agrega conceptos que pueden distraer del objetivo principal del TP:

- autenticacion
- autorizacion
- roles
- permisos
- passwords
- tokens
- sesiones

Para mantener el proyecto claro, primero se trabajara sobre el flujo principal:

1. consultar catalogos
2. registrar clientes
3. registrar inventario
4. crear rentas
5. devolver items

La seguridad puede retomarse mas adelante cuando el flujo principal ya este entendido.
